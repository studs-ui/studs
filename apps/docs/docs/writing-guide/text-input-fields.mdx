# Text Input Fields

**Purpose:** Guide users on how to enter the correct information.

**Guidelines:**

1. Avoid Punctuation: Only use punctuation if you're posing a question.
2. Use Sparingly: Placeholder text should only be used when necessary. If the input field is self-explanatory, it may not need any additional hints.
3. Avoid Redundancy: Do not repeat the input field label in the placeholder.
4. Consider Helper Text: For more detailed guidance that remains visible as the user types, use persistent helper text instead of a disappearing placeholder.
5. Case Sensitivity: Always use sentence case for placeholder text.
6. Standard Action Labels: Use "Select" for dropdowns and other standard action labels for UI interactions within input fields.

**Examples:**

Do:

- Date Input: If the label is "Date", the placeholder could be "MM/DD/YYYY".
- Dropdown: Label: "Country", Placeholder: "Select a country".
- Email Input: Label: "Email Address", Placeholder: "E.g. [jane.doe@email.com](mailto:jane.doe@email.com)".
- Password Input: Label: "Password", Placeholder: "At least 8 characters".

Don't:

- Redundancy: Label: "Country", Placeholder: "Enter your country" (redundant).
- Punctuation: Placeholder: "E.g. 01/15!" (Avoid unnecessary punctuation).
- Generic Text: Label: "Address Line 2", Placeholder: "Address continuation (if necessary)".
- Case Inconsistency: Placeholder: "e.g. 01/15" (Always use sentence case).
